# Smart-ai-system 运行备忘录

## 核心原则

**所有配置信息统一写在 `config/.env`**，启动脚本会自动加载。
**禁止在代码或启动命令中硬编码凭据。**

---

## 配置文件位置

```
/root/smart-ai-system/config/.env
```

**必须包含以下配置**（不可少）：

```env
# 飞书
FEISHU_APP_ID=cli_xxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxx

# TOS 存储（与 OpenClaw 独立）
TOS_ENDPOINT=https://tos-cn-beijing.ivolces.com
TOS_REGION=cn-beijing
TOS_BUCKET=4-ark-claw
TOS_AK=AKLTxxxxxxxxxxxxxxxx
TOS_SK=TkRGxxxxxxxxxxxxxxxx

# Gateway 回执通知
GATEWAY_URL=http://127.0.0.1:18789/webhook/notify

# Ark API Key（供应商调用，3D生成用）
ARK_API_KEY=ark-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

> ⚠️ **真实凭据值请查看 config/.env，不要将真实值写入文档**

## 正确启动 Poller

### ❌ 错误方式

```bash
# 不要这样！环境变量读不到，ARK_API_KEY 是无效的 UUID
cd /root/smart-ai-system
PYTHONPATH=src python -m ai_3d_modeling.poller
```

### ✅ 正确方式

```bash
cd /root/smart-ai-system

# 方式1: 使用启动脚本（推荐）
python scripts/run_poller.py

# 方式2: 指定参数
python scripts/run_poller.py --interval 30 --log-level DEBUG
```

**启动脚本会自动从 `config/.env` 加载所有环境变量**，无需手动 export。

## 验证 Ark API Key 是否有效

```bash
# 如果启动后 poller 报 401，先手动验证
python3 -c "
import httpx, asyncio
async def test():
    key = open('/root/smart-ai-system/config/.env').read().split('ARK_API_KEY=')[1].split()[0]
    r = await httpx.AsyncClient().get(
        'https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks',
```

## 飞书 Skill 调用流程

```
用户发飞书消息（触发 3D 建模）
    ↓
OpenClaw 接收 → AI 激活 ai-3d-modeling skill
    ↓
skill/standalone.py 接收请求
    ├── 解析意图
    ├── 创建 session + material 记录
    ├── 发送回执卡片给用户
    └── 调用 Ark API 提交供应商任务
    ↓
Poller 轮询供应商状态
    ↓
任务完成 → StorageManager 上传 TOS → 发送结果卡片
```

**Poller 必须在 Skill 之后启动**，确保轮询器运行以处理异步任务结果。

## 常见问题排查

### 401 Unauthorized (Ark API Key 无效)

**原因**: `config/.env` 中 `ARK_API_KEY` 是无效的 UUID 格式。

**解决**: 更新为有效的 Ark API Key（格式: `ark-xxxxxxxx-...`）

### 401 Unauthorized (TOS)

**原因**: `~/.tosutilconfig` 中的 AK/SK 与 `config/.env` 不一致。

**解决**: 确保两边使用相同的有效凭据。

### Poller 启动后立即退出

**原因**: Ark API Key 无效，连接 Ark API 失败。

**解决**: 参考上方「验证 Ark API Key」排查。

### 飞书回执卡片发送失败

**原因**: `GATEWAY_URL` 不正确或 Gateway 未运行。

**解决**: 确认 `GATEWAY_URL=http://127.0.0.1:18789/webhook/notify`，OpenClaw Gateway 正在运行。

## 配置加载优先级

启动脚本 `scripts/run_poller.py` 的环境变量加载顺序：

1. **已存在的环境变量**（`os.environ` 中已有的，优先级最高）
2. **`config/.env`**（作为默认值来源）

意味着：
- 如果在 shell 中 `export ARK_API_KEY=xxx`，会使用 export 的值
- 如果没有 export，则读取 `config/.env` 中的值
