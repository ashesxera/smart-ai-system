# 文档书写规范

## 文档管理规范

### 目录结构

```
/root/.openclaw/workspace/
├── smart-ai-system/           ← 本地 Git 仓库（版本管理）
│   ├── .git/
│   ├── docs/                   ← 文档目录
│   └── reference/              ← 参考资料目录
│
└── 4-ark-claw/               ← TOS 分发目录
    └── smart-ai-system/      ← 同步副本
```

### 文档存放规则

| 文档类型 | 存放位置 |
|----------|----------|
| 本地编写文档 | 先存本地 Git，再同步到 TOS |
| 用户上传文档 | 先存本地 Git，再同步到 TOS |

### Git 使用规范

#### 提交时机

- 每完成一个阶段的文档编写
- 每收到用户的重要修改
- 每天工作结束前

#### 提交信息格式

```
git commit -m "描述内容"
```

示例：
- `git commit -m "更新系统设计文档 v1.1"`
- `git commit -m "添加数据库设计"`
- `git commit -m "修正ER图关系"`

#### 版本回滚

```bash
# 查看历史
git log

# 回滚到某个版本
git checkout <commit-id> -- docs/
```

## 文档版本规范

### 版本号规则

| 版本 | 格式 | 说明 |
|------|------|------|
| v1.0 | 主版本.次版本 | 初始版本 |
| v1.1 | 主版本.次版本 | 第一次修订 |

### 变更记录

每次重要修改应在文档末尾添加变更记录：

```markdown
---

## 变更记录

### v1.1 (2026-04-20)
- 修改内容1
- 修改内容2

### v1.0 (2026-04-20)
- 初始版本
```

## 文档格式规范

### 标题层级

- `#` 一级标题：文档标题
- `##` 二级标题：章节
- `###` 三级标题：小节
- `####` 四级标题：子小节

### 表格

使用 Markdown 表格，标题行用 `-` 分隔。

### 代码块

使用 ``` 包裹，标注语言类型。

```python
# 示例代码
def hello():
    print("Hello")
```

## 文档命名规范

### 讨论记录

```
architecture-discussion-v{n}.md
```

示例：
- `architecture-discussion-v1.md`
- `architecture-discussion-v2.md`

### 设计文档

```
system-design-v{n}.md
database-design.md
```

示例：
- `system-design-v1-1.md`
- `database-design.md`
